# Dicts

*Quick reference: `(help "dicts")` -- Full documentation: this file.*


A dict is a mutable mapping from keys to values -- Python's `dict` type
exposed directly to Lisp code.  Keys can be strings, numbers, or symbols
(which are automatically converted to uppercase strings).  Values can be
any Lisp object.

---

## Creating a Dict

```lisp
(make-dict (key val) ...)
```

Key-value pairs are written directly inside `make-dict` without quoting.
Keys are **not evaluated** -- a bare symbol becomes an uppercase string key.
Values **are** evaluated at construction time.

```lisp
; Empty dict
(make-dict)
;==> (DICT)

; String keys
(make-dict ("x" 10) ("y" 20))
;==> (DICT
;       ("x" 10)
;       ("y" 20)
;    )

; Symbol keys -- stored as uppercase strings
(make-dict (width 100) (height 50))
;==> (DICT
;       ("WIDTH" 100)
;       ("HEIGHT" 50)
;    )

; Values are evaluated
(setf n 42)
(make-dict ("answer" n) ("double" (* n 2)))
;==> (DICT
;       ("answer" 42)
;       ("double" 84)
;    )
```

---

## Accessing Values

`at` retrieves a value by key.  The key argument is evaluated, so to look
up a string key you pass a string; for a symbol key you pass the uppercase
string it was stored under.

```lisp
(setf d (make-dict ("a" 1) ("b" 2) ("c" 3)))

(at "a" d)    ;==> 1
(at "b" d)    ;==> 2
```

`at` on a dict with a missing key signals an error.  Use `has-key-p` to
guard lookups when the key may be absent.

---

## Mutating a Dict

### Setting a single key

```lisp
(at-set "b" d 99)     ; returns 99, d is mutated
(at "b" d)            ;==> 99

; Equivalent setf form
(setf (at "b" d) 99)
```

### Merging two dicts

`update!` copies all key-value pairs from dict2 into dict1, overwriting
any keys that already exist in dict1.  It mutates dict1 and returns it.

```lisp
(setf base   (make-dict ("a" 1) ("b" 2)))
(setf extras (make-dict ("b" 99) ("c" 3)))

(update! base extras)
;==> (DICT
;       ("a" 1)
;       ("b" 99)   ; overwritten
;       ("c" 3)    ; added
;    )
```

## Membership Tests

```lisp
(setf d (make-dict ("x" 10) ("y" 20)))

(has-key-p   "x" d)    ;==> T
(has-key-p   "z" d)    ;==> NIL
(has-value-p 10  d)    ;==> T
(has-value-p 99  d)    ;==> NIL
(dictp d)            ;==> T
(dictp '(1 2 3))     ;==> NIL
```

---

## Iterating Over a Dict

Dicts do not have a dedicated iteration primitive, but `mapcar` over a
manually maintained key list is the standard pattern:

```lisp
(setf scores (make-dict ("alice" 95) ("bob" 87) ("carol" 91)))
(setf names  '("alice" "bob" "carol"))

(mapcar (lambda (k) (list k (at k scores))) names)
;==> (("alice" 95) ("bob" 87) ("carol" 91))
```

## Dicts and Structs

Struct instances created by `defstruct` are plain dicts under the hood,
with a reserved `"STRUCT-TYPE"` key holding the type name.  You can use
`at` and `at-set` on struct instances the same way you would on any dict,
though the accessor functions generated by `defstruct` are the preferred
interface.

---

## Type System

| Expression | Result |
|---|---|
| `(type-of d)` | `DICT` |
| `(typep d 'dict)` | T |
| `(dictp d)` | T |

---

## Quick Reference

| Expression | Meaning |
|---|---|
| `(make-dict (k v) ...)` | Create dict; keys unevaluated, values evaluated |
| `(at key dict)` | Read value at key |
| `(at-set key dict val)` | Write value at key (mutates) |
| `(setf (at key dict) val)` | Same via setf |
| `(update! dict1 dict2)` | Merge dict2 into dict1 (mutates dict1) |
| `(has-key-p key dict)` | T if key present |
| `(has-value-p val dict)` | T if value present |
| `(dictp x)` | T if x is a dict |

See also: STRUCTS for struct instances (which are dicts); SEQUENCES for
list operations.
